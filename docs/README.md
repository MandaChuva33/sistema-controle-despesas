# Estratégia de Construção do Sistema de Controle de Despesas

## Autor
Pedro Pourchet - RA: 206454-25

## Objetivo Geral
Desenvolver um Sistema de Controle de Despesas que atenda a todos os requisitos de Programação Orientada a Objetos (POO) e persistência de dados em arquivos de texto.

## Estrutura de Commits (Entregas)

O projeto foi estruturado em três commits principais, seguindo rigorosamente as especificações:

1.  **B4T01.1 (v0.0.1) - Primeiro Commit:** Foco na estrutura inicial e no menu principal.
2.  **B4T01.2 (v0.0.2) - Segundo Commit:** Foco na implementação das classes de modelo e gerenciamento (POC/MVP), aplicando os conceitos de POO.
3.  **B4T01.3 (v0.0.3) - Terceiro Commit:** Foco na documentação final, arquivos de configuração e consolidação.

## Requisitos de POO
O sistema será implementado utilizando os seguintes conceitos de POO:
- Classes e Herança (Classes Abstratas e Concretas)
- Interfaces e Polimorfismo
- Sobrecarga e Sobrescrita de Método Construtor
- Métodos e Atributos Estáticos
- Criptografia de Senhas (SHA-256)

## Persistência
Todos os dados (Despesas, Tipos de Despesa e Usuários) serão armazenados em arquivos de texto separados, conforme solicitado.
